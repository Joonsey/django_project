I made a ticket support system!

Ticketer is a ticket support system built on top of a Discord API.

What is a ticket support system?

A ticket support system is a piece of software that will allow a user (someone who works in support) to file a ticket into the system for it to be stored and reviewed at a later date.

Well that doesn't sound very complicated?

It isn't, however I made it on the Discord API for the people who uses it to have easy access and to avoid authetication and user hierarchy requirements. It's plug-and-play!

The main challenge with these software types are that they can be convoluted to use, it's usually some strange website with TONS of uncanny features and strange user interfaces.

By using Discord I avoid the majority of these challenges, it takes chat messages as commmands and arguments to commit queries and changes to a SQL database. It also has a some QoL features like: Automated ID generation, specific or dynamic queries, embedded messages, automatic chat cleaner & Database wipe function to name a few

Ok that's cool, but how does it work?

It is just a SQL database containing a table called "tickets".

The structure of the table is something like: id | date | context | author | solved

The ID is proceduraly generated, it will always be set to one higher than the highest ID in the database. the date will also be proceduraly generated based on the current time and date when the ticket is created.
Context is the only argument that has to be sent to Ticketer, it will contain the information about the problem from the customer, and what was done to try to solve it.

Now the 'Author' parameter is automatically determined by who sends the message on Discord. this can be either with their current Discord name/alias, or it can be their Discord ID.

This is where another problem arises.

The discord ID is a large number and won't really help us tell who sendt the message, if we where to recieve that. But if we use the Discord name/alias it won't be consistent, because people can simply change their name over time, and alot of people semi-frequently do.

My idea of a solution to this:

  1. Use Discord ID and map the ID's to their names before entering them to the database, using a dictionary or hashmap.
  2. Just use Discord name/alias and hope for the best.

I am no longer actively working on developing it, but if you want you can take a look at the repository

Be mindful he is not currently configured to run the database in memory. This can be changed in the database.py file
